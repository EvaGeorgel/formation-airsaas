🔍 Le Management des Risques Projet

Un risque est un événement futur possible que nous pouvons quantifier. La norme NFX50-117 distingue l'imprévu du risque réel. Dans AirSaaS, qualifier le risque permet de décider de la trajectoire.

1Identifier les risques
2Déterminer probabilité & conséquences
3Trouver des solutions
4Monitorer & Reporter

Explorez les types de risques SI classiques :

⚙️ Fonctionnel

Description : Le poids du spécifique est trop important.
💥 Impact : Dérive du périmètre et coûts.
🛠️ Action : Simplifier ou standardiser.

💻 Technique

Description : Présence d'un verrou technologique majeur.
💥 Impact : Blocage de la production.
🛠️ Action : Réaliser un POC (Proof of Concept).

🏢 Organisationnel

Description : Non-tenue des engagements (planning, charges).
💥 Impact : Retard global du projet.
🛠️ Action : Recalibrer les ressources.

👤 Client Interne / MOA

Description : Faible implication des métiers dans le projet.
💥 Impact : Solution inadaptée aux besoins.
🛠️ Action : Ateliers de co-conception forcés.

🤝 Humain

Description : Besoin de profils hyper qualifiés / Guerre des talents.
💥 Impact : Fuite des compétences clés.
🛠️ Action : Plan de backup et formation.

🛡️ Cyber / Sécurité

Description : Vulnérabilités critiques non traitées.
💥 Impact : Perte de données / Arrêt SI.
🛠️ Action : Audit de sécurité préventif.

⚖️ Légal / RGPD

Description : Changement de l'environnement juridique.
💥 Impact : Amendes ou arrêt du service.
🛠️ Action : Veille juridique constante.

🧠 Psychosociaux (RPS)

Description : Stress, harcèlement ou surcharge de travail.
💥 Impact : Burn-out et turn-over.
🛠️ Action : Transférer l'alerte aux RH.
Rappel Méthodologique : Un risque identifié dans AirSaaS doit toujours être associé à une Criticité (Bas, Moyen, Élevé) lors de la création du projet pour alerter le Sponsor.
